結合表を作成する練習問題

以下の作業は root でおこなってください。

(1) MySQLにログインし、web_school というデータベースを作成してください。
  もしも、すでに web_school という名前のデータベースがあるなら、それを削除
  してください。

  削除するときに、そのデータベースの内容を保存しておきたければ、以下のように
  してそのデータベースを保存してください。

  データベースの保存
  (a) MySQLからログアウトする。
  (b) コマンドプロンプトで、以下のコマンドを実行する。
    > mysqldump -u root -p -B web_school > web_school_bak.dump

(2) 以下の条件を満たす3つのテーブルを作成してください。
    "web_school.sql"というスクリプトを作成し、その中に記述してください。

  (a) state表

    スクリプト state.sql を読み込む。
      source state.sql
    とすることで、そこにテーブル定義を書いて、insert文を書いたのと
    同じことができる。


  (b) course表

    1. プライマリ・キーを カラム id とする。
       データ型は char型で、文字数を 3 とする。

    2. コース名のカラム名は name とする。
       データ型は VARCHAR型で、最大文字数は 50 とする。
       NULL の入力は許可しない。

    3. データとして以下の内容を含む。
       id は、"C01" のように、"C" と 2桁の数字で構成する。
       name は、英字が主だが、全角文字を使ってもよい。
       結合表.png を参考にしてINSERT文を作成し、実行する。

  (c) students表
 
    1. プライマリ・キーとして カラム id を追加する。
       データ型は整数で、自動連番とする。

    2. 名前のカラム名は name とする。
       データ型は VARCHAR型で、最大文字数を 50 とする。
       NULL の入力は許可しない。

    3. 性別のカラム名は gender とする。
       データ型は CHAR型で、文字数を 1 とする。

    4. 誕生日のカラム名は birthday とする。
       データ型は DATE型で、NULLの入力は許可しない。

    5. 出身のカラム名は state_id とする。
       データ型は CHAR型で、文字数を 2 とする。
       これは、別表の state表に対応している。  

    6. コースのカラム名は course_id とする。
       データ型は CHAR型で、文字数は 3 とする。
       これは、別表の course表に対応している。

    7. データは、結合表.pngを参考にして、
       INSERT文を実行する。


(3) 3つの表を内部結合してください。

  (a) 結合のしかた

      SELECT
        *
      FROM 基礎となる表 文字A
        INNER JOIN 結合する表1 文字B
        ON 文字A.カラム = 文字B.id
          INNER JOIN 結合する表2 文字C
          ON 文字A.カラム = 文字C.id
      ORDER BY 文字A.id ASC;

   (b) 上の結合を一度にするのではなくて、
     まず2つの表を結合させる。
     それから3つめの表を結合させる。

   (c) 表示するカラムを決めるのは、あとまわしにする。
